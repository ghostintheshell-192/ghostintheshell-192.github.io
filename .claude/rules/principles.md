# Development Principles

## Core Values

1. **Simplicity First**: Choose the simplest solution that works
2. **Documentation-Driven**: Document decisions before implementation
3. **Security by Default**: Never compromise on security
4. **Fail Fast**: Validate early, catch errors at compile time when possible
5. **Explicit Over Implicit**: Clear intent over clever code

## Code Quality

- **Readability**: Code is read more than written - optimize for clarity
- **Maintainability**: Future developers (including you) will thank you
- **Testability**: Design for testing from the start
- **Performance**: Optimize when needed, not prematurely

## Decision Making

- **Document Significant Decisions**: Use ADRs for architectural choices
- **Consider Trade-offs**: Every decision has costs - document them
- **Reversibility**: Prefer reversible decisions when possible
- **Data Over Opinions**: Measure, don't guess

## Collaboration

- **Clear Communication**: Over-communicate rather than under-communicate
- **Assume Good Intent**: Everyone is trying their best
- **Knowledge Sharing**: Document and share learnings
- **Feedback Culture**: Give and receive feedback constructively
