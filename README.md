# Project Scaffold Template

This directory contains the template structure for initializing new projects with Claude Code configuration.

## Structure

```
.claude/
├── CLAUDE.md                    # Project entry point with navigation
├── settings.json                # Claude Code permissions and hooks
├── commands/
│   └── handoff.md              # Session handoff command
├── skills/
│   └── session-handoff/
│       └── SKILL.md            # Session handoff skill
└── rules/                       # Auto-loaded project rules
    ├── overview.md             # Project overview (customize for project)
    ├── coding-standards.md     # Coding standards (auto-generated by tech stack)
    ├── principles.md           # Development principles
    ├── preflight-checks.md     # Pre-work checklist
    └── workflow.md             # Git workflow

.memory-bank/                    # Session handoff notes
└── sessions/                    # JSONL transcript files

.scripts/                        # Automation scripts
└── session-archive.py          # Archives session handoffs
```

## Usage

When initializing a project with DevDash:

1. **Copy scaffold structure** to project root
2. **Customize placeholders** in `.claude/CLAUDE.md` and `.claude/rules/overview.md`:
   - `ghostintheshell-192.github.io` - Project name
   - `Personal WebPage` - Short description
   - `HTML` - Tech stack details
   - ````
ghostintheshell-192.github.io/
```
` - Directory structure
   - `` - Coding standards for detected language
3. **Create .development/** structure with:
   - `CURRENT-STATUS.md`
   - `ARCHITECTURE.md`
   - `specs/`, `tech-debt/`, `reference/`, `archive/`, `scripts/`
4. **Create .personal/** for private notes (not tracked)
5. **Create docs/** for public documentation

## Template Placeholders

DevDash will replace these placeholders when initializing:

- `ghostintheshell-192.github.io` - from project directory name or user input
- `Personal WebPage` - from user input
- `HTML` - auto-detected or user input
- ````
ghostintheshell-192.github.io/
```
` - auto-generated from directory scan
- `` - selected from language-specific templates

## Coding Standards Templates

DevDash includes language-specific coding standards templates in:
`rsrc/legacy-workspace-rules/scaffold-rules/coding-standards/`

These are used to populate `` based on detected tech stack:
- `csharp-dotnet.md` - C# / .NET projects
- `python.md` - Python projects
- `javascript-typescript.md` - JS/TS projects
- `flutter-dart.md` - Flutter/Dart projects
- `c-cpp.md` - C/C++ projects

## Official Claude Code Pattern

This scaffold follows Claude Code's official `.claude/rules/` pattern:
- All `.md` files in `.claude/rules/` are **automatically loaded** as project instructions
- No `@includes` needed - Claude Code discovers and loads them
- Rules can be organized in subdirectories
- Path-specific rules can be added with YAML frontmatter
